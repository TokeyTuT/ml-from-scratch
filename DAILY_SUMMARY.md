- 明确当前项目先实现经典的 CART 二叉分类树，不同时扩展回归树。
- 设计 `Node` 节点结构，用于保存样本数量、划分特征、划分阈值、左右子树和节点预测类别。

- 加入 `max_depth`、`min_sample_split` 和 `min_sample_leaf` 等控制参数，为限制树的复杂度和缓解过拟合做准备。

### 遇到的问题与思考

- ID3、C4.5 和 CART 的特征选择标准不同，需要先分清信息增益、信息增益率和基尼指数各自解决的问题，再决定代码采用哪一种方案。
- 连续特征不能直接按照类别取值建立多叉树，因此 CART 需要对候选阈值进行遍历，并将数据划分成左右两个子集。
- 决策树的建树过程天然适合使用递归，但必须处理好纯节点、空子集、最大深度、最小划分样本数和找不到有效划分等停止条件，否则容易出现无限递归或无效节点。
- 叶子节点需要保存当前样本中的多数类别，即使后续不能继续划分，也能够给出预测结果。
- 当前代码属于第一版逻辑实现，核心结构已经搭建，但候选阈值生成、基尼概率计算、标签编码、递归停止条件和预测数组构造仍需要通过运行与测试逐项检查。

### 今日收获

- 理解了决策树训练的核心就是反复寻找能够让子节点更纯的划分，并递归地构建左右子树。
- 掌握了熵越大表示数据越混乱、基尼值越小表示数据越纯的直观含义。
- 理解了信息增益率是对信息增益偏好多值特征问题的一种修正，而 CART 则使用二叉划分和基尼指数形成另一套建树方案。
- 将递归、树节点、数据切片和多数表决结合到一个完整模型中，对决策树从公式到代码的映射有了初步认识。

### 实验结果

- 决策树演示中的 14 项行为和异常检查全部通过。
- Titanic 数据共包含 891 条训练记录，其中 342 人生存，总体生存率为 38.38%。
- 数据中 `Age` 缺失 177 条，`Cabin` 缺失 687 条；年龄使用训练部分的中位数进行填充，并额外保留年龄缺失标记。
- 构造了舱位等级、性别、年龄、家庭人数、是否独自出行和是否有客舱记录等数值特征。
- 手写决策树的训练集准确率为 83.45%，验证集准确率为 80.90%，高于 61.80% 的多数类基线。

- 最终模型深度为 5，共有 43 个节点和 22 个叶子节点；树中使用较多的特征包括年龄、舱位等级、家庭人数和性别。
- 对 Kaggle 测试集预测出 153 名乘客生存，预测生存率为 36.60%。

### 遇到的问题与解决

- 第一版基尼不纯度错误地使用了类别值而不是类别出现概率。修正后先统计每个类别的样本数，再计算 `1 - sum(p_k^2)`。
- 第一版候选阈值和右子集索引存在错误。修正后使用相邻唯一特征值的中点，并通过布尔掩码统一划分左右数据。
- 递归函数直接返回 `None` 会破坏树结构和预测路径，因此改为先创建保存多数类的叶子节点，只有满足条件时才继续补充左右子树。
- `min_samples_leaf` 不能等到子树建完后再处理，需要在选择划分时提前排除会产生过小叶子的候选阈值。
- Titanic 中同时存在缺失值和类别字段，不能直接交给当前只接收数值特征的决策树，因此单独实现预处理流程，将性别等类别转换为数值，并对年龄缺失进行填充和标记。
- 为避免验证集信息泄漏，年龄中位数只使用开发训练集计算，再用于开发训练集、验证集和 Kaggle 测试集。
- 单看准确率无法完整评价 Titanic 生存预测，因此同时计算精确率、召回率、混淆矩阵，并与多数类基线比较。

### 遇到的问题与思考

- Boosting 和 Bagging 都会组合多个模型，但模型之间的关系不同：Boosting 的学习器前后依赖，需要逐轮训练；Bagging 的学习器相互独立，可以并行训练。
- AdaBoost 同时存在两套权重：样本权重决定下一轮重点学习哪些数据，模型权重决定最终投票时更相信哪个弱分类器。
- 树桩不仅要寻找最佳特征和阈值，还要比较阈值左右两种分类方向，否则可能错过加权错误率更低的规则。
- AdaBoost 的标准二分类推导通常要求标签使用 `-1` 和 `+1`，后续需要在公开接口中验证标签或自动完成编码与还原。
- 当弱分类器错误率为 0 或 1 时，模型权重公式可能出现除零或无穷大；错误率接近 0.5 时，该分类器也几乎没有提升价值，需要增加数值稳定性和提前停止逻辑。
- 当前实现依赖 `X` 为二维 NumPy 数组，尚未系统检查空数据、非法维度、NaN、特征数量不一致及未训练预测等情况。
- GBDT 与 AdaBoost 都属于 Boosting，但优化思路不同：AdaBoost 直接调整样本权重，GBDT 则逐轮拟合当前损失的负梯度，在平方误差下可以直观理解为拟合残差。

### 今日收获

- 建立了集成学习的整体框架，能够从学习器是否独立、数据如何采样、错误如何反馈以及最终如何组合等角度区分 Bagging 与 Boosting。

- 掌握了 AdaBoost 从初始化等权样本、训练弱分类器、计算模型权重、更新样本权重到加权投票的完整流程。
- 将 AdaBoost 数学公式映射到了 NumPy 实现，对布尔掩码、加权错误率和指数权重更新有了更具体的认识。
- 明确了后续学习顺序：先巩固 AdaBoost，再实现回归树和 GBDT，最后在梯度提升基础上理解 XGBoost 的二阶梯度与正则化。

## Day 8 — 2026 年 7 月 19 日：XGBoost 原理推导与初步实现

### 今日完成

- 正式提交 GBDT 的非线性回归分析脚本，并将前一天的学习内容补充到每日总结中。
- 开始系统学习 XGBoost，理解它是在梯度提升框架上加入二阶梯度信息和树复杂度正则化的一种提升树算法。
- 在 `NOTE.md` 中补充 XGBoost 的数学推导，从带正则项的目标函数出发，对每轮新增树的损失进行二阶泰勒展开。
- 推导叶子节点的最优权重：
  \[
  w_j^*=-\frac{G_j}{H_j+\lambda}
  \]
  其中 \(G_j\) 和 \(H_j\) 分别表示叶子内一阶梯度与二阶梯度之和。
- 推导节点分裂收益，理解只有收益大于 0 时才值得继续划分；同时明确 `gamma` 用于惩罚新增叶子，`reg_lambda` 用于限制叶子输出幅度。
- 实现 XGBoost 专用回归树 `XGBoostTree`，使节点划分不再使用普通平方误差，而是根据梯度统计量计算分裂收益。
- 为 XGBoost 树实现特征遍历、相邻唯一值中点阈值、最优收益搜索、递归建树和单样本预测。
- 实现基于一阶、二阶梯度的叶子权重计算，并将该权重作为叶子节点的预测输出。
- 初步实现 `XGBoost` 回归模型，使用目标均值作为初始预测，按顺序训练多棵 XGBoost 树。
- 针对平方误差设置一阶梯度为“当前预测减真实值”、二阶梯度为 1，并在每轮训练后更新当前梯度。
- 实现学习率控制下的多棵树预测累加，并修复 XGBoost 中部分梯度更新和冗余代码问题。

- `11_xgboost/` 通过 Python 语法编译检查。
- 使用固定随机种子生成 240 个双特征非线性回归样本，其中 180 个用于训练、60 个用于测试。
- 使用 30 棵最大深度为 2 的树，设置学习率为 `0.3`、`reg_lambda=1.0`、`gamma=0.0`。
- 模型成功训练出 30 棵树，训练 MSE 为 `0.3533`。
- 测试 MSE 为 `0.9980`，测试 R² 为 `0.9140`。
- 同一测试集上的均值基线 MSE 为 `11.7735`，说明当前实现能够从简单合成数据中学习到明显的非线性规律。

### 遇到的问题与思考

- GBDT 的回归树通常直接拟合残差，而 XGBoost 树需要接收每个样本的一阶梯度和二阶梯度，并以目标函数下降量作为划分依据。
- XGBoost 叶子输出不是目标值或残差的简单平均，而是由叶子内梯度总和、Hessian 总和和 L2 正则项共同决定。
- `gamma` 与 `reg_lambda` 的作用不同：`gamma` 直接提高节点继续分裂的门槛，`reg_lambda` 则压缩叶子权重并影响分裂收益。
- 对平方误差而言 Hessian 恒为 1，但保留 Hessian 数组能够让树结构以后扩展到其他可二阶求导的损失函数。
- 每轮新增树输出的是对当前预测的修正量，因此训练阶段梯度更新与预测阶段树结果累加必须使用一致的学习率。
- 当前实现尚未验证空数据、非法维度、NaN、未训练预测、参数范围和训练特征数量变化等情况。
- `predict` 当前按 `n_estimators` 遍历模型，后续若加入提前停止，更稳妥的方式是直接遍历实际保存的 `estimators_`。
